GAMEPLAY FLOW (CLI)
1. `seed` loads demo blueprints.
2. `blueprints` explore assets by `category`/`rarity`.
3. `sources <blueprint_key>` inspect crafting routes.
4. `forge <json>` produce an asset:
   - selects a crafting source (`crafting_source_id` optional)
   - applies source + slot bonuses
   - auto-fills slots from explicit `addons` then mythic materials
   - applies slot magic rolls and optional lock state
5. Optional: `program`, `slot_lock`, `slot_reroll`, `slot_upgrade`.
6. Optional: `roles`/`optimize`/`benchmark`/`eval_stats` for balancing decisions.
7. Optional: `mint` and `port` for cross-system handoff.

FORGE STATE MODEL (short)
- `AssetBlueprint` now stores:
  - `category` grouping
  - `crafting_sources` (with process steps, materials, tools, requirements, multipliers, risk, yield, enhancements)
  - `addon_slots` (per-slot `slot_id`, `slot_type`, compatibility, roll bounds).
- `ForgedAsset` tracks:
  - `crafting_source_id`
  - `applied_enhancements`, `magic_effects`
  - `addon_slots_filled` containing slot source (addon/material/empty), roll metadata, lock status
  - mythic material usage and provenance
  - optional `NFToken` after mint.

SLOT MAGIC FLOW
- Forge defaults come from slot schema:
  - `default_magic_rolls` (default 1)
  - `max_magic_rolls` (default at least 3 unless raised)
- Supported operations:
  - `slot_roll_seed` + `slot_rolls` allow deterministic, per-slot roll budgets.
  - `slot_lock` prevents `slot_reroll` from changing that slot.
  - `slot_upgrade` increases `magic_roll_count` up to `max_magic_rolls` then rerolls that slot with same cap.

SCORING/OPTIMIZATION MODEL
- Role profiles from `item_optimization.py`:
  - `dps`, `tank`, `support`, `controller`, `balanced`
- Score shape:
  - stat grouping/alias map → role weighted score
  - rarity profile multipliers/bonuses (`stat_multiplier`, `enchant_multiplier`, `rarity_bonus`, `trigger_cap`)
  - enchantment scoring with trigger/group/condition analysis
  - creativity score for stat-family variety and meta-penalty
  - `total_score` combines role score + creativity + enchantment + rarity bonus (with meta clamp).
- `evaluate` output includes:
  - `role_score`, `creativity_score`, `enchantment_score`, `rarity_bonus`, `total_score`
  - `group_summary`, `enchantments`, `creativity`, `stats_count`, `family_count`.
